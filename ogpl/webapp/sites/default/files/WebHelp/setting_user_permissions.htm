Setting User Permissions

A Super Administrator can view the list of user roles and grant permissions to each user role. You can control what users do on the Open Government Platform (OGPL) Control Panel application by granting or denying permissions. Each user role has its own set of permissions. For example, you can grant the permission to create and delete a content record to the CMS Administrators user role, but not to the Content Creator user role.

To set user permissions for different roles:

  1. In the Drupal menu, point to User management and click Permissions.
    The Permissions page appears.

  2. Under the appropriate user role column, select the check box for the permission that you want to grant to that user role. For example, if you want to allow the CMS Administrator role to create new users, under the CMS Administrator column, select the create users check box.

  3. Scroll down until you reach the end of the page, and then click Save permissions.
    The permissions for the user roles are updated.
