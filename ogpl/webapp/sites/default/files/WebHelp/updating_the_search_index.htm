Updating the Search Index

Indexing collects, parses, and stores data to facilitate fast and accurate information retrieval. The purpose of storing an index is to optimize speed and performance in finding relevant content for a search query. The cron automatically updates the search index with the updates made to the content in the OGPL Web site. You can create the search index by running the cron job. For more information about the cron job, see <topic for cron>. You can re-index all the content and overwrite the existing search index to update the search index with the latest content on the OGPL Web site.

To re-index all the content:

  1. In the navigation menu, point to Configurable Items, Manage Header Content, and then click Manage Global and Advance Search.
    The Settings page appears.

  2. Click the Search Index tab.
    The Apache Solr page appears.

  3. Click Re-index all content.
    A page that displays the confirmation message appears.

  4. Click Re-index.
    The Apache Solr Page appears.

  5. In this page, click the running cron link that appears at the top of the page to run the cron.
    All the content of the OGPL Web site is sent to the server for re-indexing and the index is updated accordingly.
