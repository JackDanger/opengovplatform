Understanding User Roles

Super Administrator

Super Administrator is the highest authority across the entire OGPL Control Panel. A super administrator creates all the administrator roles (such as CMS Administrator and VRM Administrator), and adds users to these roles. The super administrator also creates menus to be displayed on the OGPL Web site.

Only the super administrator can delete an OGPL Web site. 

 

CMS Administrator

The CMS Administrator manages the complete content management workflow, from creating users and adding user roles for the CMS workflow, configuring various Web site options required for the OGPL Web site, to generating reports.

The CMS Administrator can perform the following tasks:

  • Add, edit or delete a user account. For more information, see Managing Users.

  • View or edit the currently configured agencies. For more information, see Managing Agencies.

  • View the themes currently available for the OGPL Web site. For more information, see Viewing the List of Themes.

  • Enable a theme for a user role. For more information, see Enabling a Theme.

  • Specify a default theme for the OGPL Web site. For more information, see Selecting a Default Theme.

  • Configure the global settings for all the themes. For more information, see Configuring Global Settings for Theme.

  • Manage the Blocks content. For more information, see Managing Blocks Content.

  • Configure the OGPL Web site parameters. For more information, see Configuring the Site Information.

  • Create or edit content to be published to the OGPL Web site. For more information, see Managing Content.

  • Configure the accessibility settings, such as the text resize feature. For more information, see Managing Accessibility.

  • Configure the search parameters across the entire OGPL Web site. For more information, see Managing Global/Advanced Search.

  • Configure the Blocks to display generic content on the OGPL Web site. For more information, see Managing Blocks Content.

  • Configure the menus and sub-menus to display on the OGPL Web site. For more information, see Managing Menus.

  • Configure the Captcha settings for various OGPL Web pages. For more information, see Managing Captcha.

  • Configure the Homepage Tabs. For more information, see Managing Homepage Tabs.

  • Configure the RSS feed settings for the OGPL Web site. For more information, see Managing RSS Feeds.

  • View CMS Reports, such as Error logs and Access Statistics. For more information see, CMS Reports.

  • Upload Reference Documents. For more information, see Uploading Reference Documents.

  • Contact the Super Administrator of OGPL Control Panel application. For more information, see Contacting the Super Administrator.

 

Content Creator

Content Creators update the content that the CMS Administrator has created and assigned for update. After updating the content, Content Creator sends it to the Moderator for review. If the content is inappropriate, the moderator, the publisher or the CMS Administrator can assign the content back to the Content Creator for further update.

A Content Creator can perform the following tasks:

  • View the list of content assigned by the CMS Administrator for update. For more information see, Viewing the Content List.

  • Filter the content list according to content type and workflow step. For more information see, Filtering the Content List.

  • Update the content assigned by the CMS Administrator. For more information see, Updating the Content.

  • Preview the content as it will appear on the OGPL Web site when it is published. For more information see, Previewing the Content.

  • Send the content for Moderator review. For more information see, Sending the Content for Approval.

  • View revisions to the content. For more information see, Viewing the Revision History of the Content.

  • Track the updates performed by the other users. For more information see, Tracking the Revisions to the Content.

  • View CMS Reports, such as Error logs and Access Statistics. For more information see, CMS Reports.

  • View VRM Reports. For more information, see Viewing VRM Reports.

  • Upload Reference Documents. For more information, see Uploading Reference Documents.

  • Contact the Super Administrator of OGPL Control Panel application. For more information, see Contacting the Super Administrator.

 

Moderator

CMS moderators can view the list of content that the CMS Administrator has assigned for review. If the content is appropriate, the CMS Moderator approves the content. If the content is inappropriate, the CMS Moderator must assign the content back to the Content Creator.

The CMS Moderator can perform the following tasks:

  • View the list of content assigned for review and approval. For more information see, Viewing the Content List.

  • Filter the content list according to a specific content type and workflow step. For more information see, Filtering the List of Content.

  • Preview content. For more information, see Previewing the Content.

  • Approve content. For more information see, Approving Content.

  • Send content back to the Content Creator. For more information see, Sending the Content Back to the Content Creator.

  • View content revisions. For more information see, Viewing the Revision History of a Content.

  • Track content updates. For more information see, Tracking the Revisions to the Content.

 

Publisher

CMS Publishers can view the list of content that the CMS Moderators have submitted for review and approval. If the content is appropriate, the Publisher approves the content and publishes it on the OGPL Web site. If the content is inappropriate, the Publisher assigns the content back to the Content Creator or the Moderator, with appropriate review comments.

The CMS Publisher can perform the following tasks:

  • View the list of content assigned for review and approval. For more information see, Viewing the Content List.

  • Filter the content list according to content type and workflow step. For more information see, Filtering the Content List.

  • Preview the content as it will appear on the OGPL Web site when it is published. For more information see, Previewing the Content.

  • Send the content for Moderator review. For more information see, Sending the Content for Approval.

  • Approve content. For more information see, Approving the Content.

  • View revisions to the content. For more information see, Viewing the Revision History of the Content.

  • Track the updates performed by the other users. For more information see, Tracking the Revisions to the Content.

  • View CMS Reports, such as Error logs and Access Statistics. For more information see, CMS Reports.

  • View VRM Reports. For more information, see VRM Reports.

  • Upload Reference Documents. For more information, see Uploading Reference Documents.

  • Contact the Super Administrator of OGPL Control Panel application. For more information, see Contacting the Super Administrator.

 

VRM Administrator

The VRM Administrator manages the feedback workflow. The VRM Administrator can view the list of all the feedback received on the OGPL Web site. The VRM Administrator can perform the following tasks for each feedback:

  • View the list of feedback. For more information, see Viewing the List of Feedback.

  • Assign a feedback to the Program Management Office, Point of Contact, or VRM Administrator. For more information, see Assigning feedback.

  • Reassign feedback that the Program Management Office, VRM Administrator or Point of Contact reverted. For more information, see Reassigning the feedback.

  • Forward a feedback to the Program Management Office or Point of Contact. For more information, see Forwarding the Feedback.

  • Reply to a feedback. For more information, see Replying to the feedback.

  • Add notes to a feedback. For more information, see Adding Notes to a feedback.

  • Review feedback. For more information, see Reviewing a feedback.

  • Archive a feedback. For more information, see Archiving a feedback.

  • Close a feedback. For more information, see Closing a feedback.

  • Print feedback details. For more information, see Printing Feedback Details.

  • Update the status of various feedback simultaneously. For more information, see Updating the Status for Various Feedback Simultaneously.

  • View workflow history of a feedback. For more information, see Viewing the Workflow History for Feedback.

 

The VRM Administrator can also perform the following administrative tasks:

  • Add, edit, delete, and track a feedback category. For more information, see Managing Feedback Categories.

  • Add, edit, delete, and track a feedback action status. For more information, see Managing the Action Status for a Feedback.

  • Set, modify, and track the delay time for responding to a feedback. For more information, see Managing Feedback Delay Time.

 

Program Management Office (PMO)

The Program Management Office (PMO) is a team of people who are responsible for implementing the feedback that the VRM Administrator assigned.

The PMO can perform the following tasks for each feedback:

  • View the list of feedback that the VRM Administrator assigned. For more information, see Viewing the List of Feedback

  • Update the feedback action of a feedback. For more information, see Selecting Action Status

  • Reply to a feedback. For more information, see Replying to Feedback.

  • Review feedback. For more information, see Reviewing a feedback.

  • Forward a feedback to another Program Management Office or Point of Contact. For more information, see Forwarding  Feedback.

  • Add notes to a feedback. For more information, see Adding Notes to Feedback.

  • Revert a feedback to the VRM Administrator. For more information, see Reverting Feedback.

  • Print feedback details. For more information, see Printing Feedback Details.

  • View feedback workflow history. For more information, see Viewing the Workflow History for Feedback.

Note: The PMO cannot perform any administrative tasks.

 

Point of Contact (POC)

The Point of Contact (POC) is a representative of an agency that works with a government office. The POC can perform the following tasks for each feedback:

  • View the list of feedback that the VRM Administrator assigned. For more information, see Viewing the List of Feedback

  • Update the feedback action of a feedback. For more information, see Selecting Action Status

  • Reply to a feedback. For more information, see Replying to Feedback.

  • Review feedback. For more information, see Reviewing a feedback.

  • Forward a feedback to another Program Management Office or Point of Contact. For more information, see Forwarding  Feedback.

  • Add notes to a feedback. For more information, see Adding Notes to Feedback.

  • Revert a feedback to the VRM Administrator. For more information, see Reverting Feedback.

  • Print feedback details. For more information, see Printing Feedback Details.

  • View workflow history of a feedback. For more information, see Viewing the Workflow History for Feedback.

 

Note: The POC cannot perform any administrative tasks.
