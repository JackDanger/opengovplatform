Configuring General Settings for Captcha

You can configure the CAPTCHA settings, such as length, font settings, distortion, and noise settings, by performing the following steps:

  1. In the navigation menu, point to Configurable Items, and then click Manage Captcha.
    The General Settings tab displays the global settings for the CAPTCHA that you selected.

  2. Configure the appropriate CAPTCHA settings. The following table lists each field and its description.

Table: Description of fields in the General Settings tab on CAPTCHA page

Field

Description

Form protection

Select the CAPTCHA type that you want to implement for each listed OGPL Web page.

To add CAPTCHA to a new Web page, use the text box at the bottom of the table or click Add CAPTCHA administration links to forms.

Default challenge type

Click the default CAPTCHA type that you want to use on the OGPL Web site. If you want to specify a particular CAPTCHA type for a specific OGPL Web page, click the appropriate option in the Challenge type list.

Add CAPTCHA administration links to forms

If you want to allow the OGPL Web site users with the "administer CAPTCHA settings" permission to see CAPTCHA administration links on all forms, except on administrative pages, select this check box.

Allow CAPTCHAs and CAPTCHA administration links on administrative pages

If you want to add CAPTCHAs on the administrative pages of the OGPL Web site, select this check box. By default, CAPTCHAs are disabled on administrative pages to avoid processing overhead. In some situations, such as demo sites, you can allow CAPTCHAs on administrative pages.

CAPTCHA placement caching

The structure of the OGPL Web pages does not change frequently and the position of the CAPTCHA in each of the configured Web pages are stored in the cache. You must clear this cache when you change the structure of a Web page or when you are rebuilding the OGPL Web site.

If you want to clear the stored values for CAPTCHA placement on the OGPL Web site, click Flush the CAPTCHA placement cache.

Add a description to the CAPTCHA

If you want to provide a description, such as text to explain the purpose of using CAPTCHA to the visitor, select this check box.

CAPTCHA description

Enter a description for the CAPTCHA. For example, This question verifies that you are a human visitor. This prevents spam submissions.

Default CAPTCHA validation

If you want the CAPTCHA validation to be case-sensitive, click Case sensitive validation: the response has to exactly match the solution.

If you do not want the CAPTCHA validation to be case-sensitive, click Case insensitive validation: lowercase/uppercase errors are ignored.

Note: You can override or ignore this default setting in individual CAPTCHA types.  

Persistence

Click this option to specify the CAPTCHA behavior during the rest of the session, after the user successfully responds to one CAPTCHA challenge.

For example, if you want to display the CAPTCHA regardless of whether the user has successfully responded earlier, click Always add a challenge.

Log wrong responses

If you want to store the information about wrong responses in the database log, select this check box.

 

  1. Click Save configuration.
    The general settings for CAPTCHA configuration are updated in the OGPL Control Panel application.
