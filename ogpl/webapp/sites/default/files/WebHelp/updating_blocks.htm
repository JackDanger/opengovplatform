Updating Blocks

A Block is a generic content area that is used to provide various kinds of information on the OGPL Web site. When a new content needs to be added on the OGPL Web site, the CMS Administrator creates a block, and assigns the block to you. You must update it with the appropriate content and send it to the Moderator for review and approval.

The following image illustrates how a Block appears on the OGPL Web site.

To update a block:

  1. In the navigation menu, click the Home link.

  2. In the Content List section, click the Edit this record icon  for the Block that you want to update.
    The Edit page appears.

  3. In this page, update the appropriate information in each field. The following table lists each field and its description.

Table: Description of fields in the Create Block page

Field

Description

Title (Required)

Update the title of this block.

Body

Update the content for this block.

Content Management

If you want to send the content to the Moderator, click Moderator Review.

If you want to retain the content with you for further update, click Create/Edit Content.

Comment

Enter additional information about this content. This information appears while viewing workflow history of this content.

Revision information – Log message

Enter additional information about the updates that you made to this block. This message appears while viewing the revision history for this content.

Enter the Expiry Date

Update the date when the information in this block is outdated.

The content remains on the OGPL Web site until it is unpublished  or deleted by the Super Administrator.

Notify Before (Days)

Update the number of days before the expiry date when you want to receive a notification about the expiry of the content.

 

    1.  Click Save. The content record is updated.

Note: If you want to see how the content looks on the OGPL Web site before the content is published, click Preview.
