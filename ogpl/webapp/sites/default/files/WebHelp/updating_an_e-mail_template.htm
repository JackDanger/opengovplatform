Updating an E-mail Template

You can update the details of an existing e-mail template, such as name of the template, subject of the template, and other template details.

The following e-mail templates can be updated:

  • Template for Forward Email

  • Template for Reply Email

  • Template for Email Notification for Content Expiry

To update an e-mail template:

  1. In the Drupal menu, point to Site building, and then click Previewable email templates.
    The Previewable Email Templates page appears.

  2. In the Operations column, click the edit link for the e-mail template that you want to update.
    The Edit Template_Name Template page appears where Template_Name is the name of the template that you selected.

  3. Update the appropriate information in each field. The following table lists each field and its description.
    Table: Description of fields in the Edit Template_Name Template page

Columns

Description

Name

Update the machine name for this e-mail template. Drupal uses this name in URLs and API calls. Ensure that the name:

  • is in lower case.

  • is less than or equal to 32 characters.

  • does not have any special characters, except numbers and underscores.

Title

Update the descriptive title for this e-mail template. For example, if you want to rename a template that the PMO must use while forwarding a feedback, enter Template for Forward Email - PMO.

Subject

Update the text that must appear in the subject line of this e-mail template. You can also use tokens provided in the Replacement patterns section.

Body

Update the text that must appear in the body of this e-mail template. You can also use tokens provided in the  Replacement patterns section.

Replacement Patterns

Displays the list of available Drupal tokens that you can use with this template.

 

  1. Click Submit.
    The template is updated.

Note: In addition to the Drupal tokens, you can use the following custom tokens that are available for forwarding a feedback and replying a feedback e-mail template:

  • The custom token [assignee-comments] is replaced with the comments that the feedback owner has entered while forwarding or replying a feedback.

  • The custom token [assignee-subject] is replaced with the subject that the feedback owner has entered while forwarding or replying a feedback.
