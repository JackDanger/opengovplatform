Managing Global and Advanced Search

The visitors of the OGPL Web site can use the global and advanced search functionality to search for a specific dataset or document. The OGPL Web site uses Apache Solr as the search server to scan the search index and display the results that matches the search criteria. You must execute the cron job to index all the content on the OGPL Web site to ensure fast and accurate information retrieval during search operations. For more information on cron, see Managing Cron.  For more information on indexing, see Updating Search Index.

You can perform the following tasks to make the content on the OGPL Web site searchable:

  • Configure the Apache Solr search server. For more information see, Configuring the Apache Solr Server.

  • Index all the content on OGPL Web site by executing the cron job. For more information, see Updating Search Index.

  • Enable filters to create search filter blocks. For more information, see Enabling Search Filters.

  • Give bias to certain properties of the content when ordering the search results. For more information, see Content Bias Settings.

  • Specify weightage for the content fields that you want to give priority while searching. For more information, see Search fields.
