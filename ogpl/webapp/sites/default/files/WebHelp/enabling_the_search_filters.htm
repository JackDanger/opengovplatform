Enabling the Search Filters

You can enable various search filters, such as Filter By Catalog Type, Filter By Agency Name, Filter By Sub-Agency Name, to group the search results. When you enable a search filter, Drupal creates a block that appears on the Blocks page and you can place this block in the Search Filter region that appears on the left pane of the Search page and the Data Catalogs page. The Apache Solr server scans the content for the searched keyword and groups the results according to these filters. For example, if a visitor searches for the Raw Data available on the OGPL Web site. Out of 15 matches, 5 are found under the Agency named Defense Department, the left pane of the search page displays the agency named Defense Department and the number of result found (5) for that agency under the Filter By Agency Name section.

The following image illustrates how the Search Filters appear on the OGPL Web site.

To set the search filter blocks that must be available on the search page and the Data Catalogs page, perform the following steps.

  1. In the navigation menu, point to Configurable Items, Manage Header Content, and then click Manage Global and Advance Search.
    The Settings page appears.

  2. Click the Enabled Filters tab.
    The Apache Solr page appears and displays various search filters.

  3. Select the check box for each filter that you want to display on the search page.

  4. Click Save.
    The filters that you selected appear on the left pane of search page.
