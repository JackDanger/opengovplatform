Viewing the VRM Delay Analysis Report

The VRM Delay Analysis report lists the username of the feedback owners who did not review the feedback or send the feedback reply within the default delay time. This report enables you to analyze and track the number of late reviews or late feedback replies for each feedback owner.

The following table lists each column of the VRM Delay Analysis report and its description.

Table: Description of fields in the VRM Delay Analysis report

Title

Description

Assigned To

Displays the username of the feedback owner.

Note: If a feedback is not assigned and the default delay time has lapsed, the record column displays None.

Created Date

Displays the date when a visitor submitted the feedback.

Reply/Reviewed Date

Displays the date when the feedback owner reviewed the feedback or replied to the visitor who submitted the feedback.

Note: If the feedback owner has not reviewed or sent a reply for this feedback, the record column appears blank.

You can also view the VRM Delay Analysis report for a specific assignee and duration by performing the following steps:

  1. In the Assigned To list, click the username of the feedback owner for whom you want to generate this report.

  2. In the Status list, click the status of the feedback for which you want to view this report.

  3. In the From date box, click the start date of the duration within which you want to view this report.
    For example, if you want to view the VRM Delay Analysis report between January 1, 2012 and January 31, 2012, click January 1, 2012.

  4. In the To date box, click the end date of the duration within which you want to view this report.
    For example, if you want to view the VRM Delay Analysis report between January 1, 2012 and January 31, 2012, click January 31, 2012.

  5. Click Submit.
    The VRM Delay Analysis report for the selected criteria appears.
