Updating Pages

When a CMS Administrator assigns a Page to you, you must update it with the appropriate content and send it to the Moderator for review and approval.

The following image illustrates how a Page appears on the OGPL Web site.

To update the content for a page:

  1. In the navigation menu, click the Home link.

  2. In the Content List section, click the Edit this record icon   for the Page that you want to update. The Edit page appears.

  3. Update the appropriate information in each field. The following table lists each field and its description.

Table: Description of fields in the Edit page

Field

Description

Title

Update the title of this page.

Body

Update the content for this Page.

Content Management

If you want to send the content to the Moderator, click Moderator Review.

If you want to retain the content with you for further update, click Create/Edit Content.

Comment

Enter additional information about this page. This information appears while viewing workflow history of this content.

Revision information – Log message

Enter additional information about the updates that you made to this page. This message appears while viewing the revision history for this content.

Enter the Expiry Date

Update the date when the information in this page is outdated.

The content remains on the OGPL Web site until it is unpublished or deleted by the Super Administrator.

Notify Before (Days)

Update the number of days before the expiry date when you want to receive a notification about the expiry of the content.

    1. Click Save. The content record is updated.

Note: If you want to see how the content looks on the OGPL Web site before the content is published, click Preview.
