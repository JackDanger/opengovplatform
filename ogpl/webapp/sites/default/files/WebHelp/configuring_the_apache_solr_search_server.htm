Configuring the Apache Solr Server

You can configure the Apache Solr server and specify the number of results that must appear on the search page, the number of items to index for each cron job, and other additional settings, such as help messages, alternative results that must appear when the Apache Solr server is unavailable or there is no match found for the searched keyword.

To configure Apache Solr server:

  1. In the navigation menu, point to Configurable Items, Manage Header Content, and then click Manage Global/Advance Search.
    The Apache Solr page appears.

  2. Enter appropriate information in each field. The following table lists each field and its description.

Table: Description of fields in the Apache Solr page

Field

Description

Solr host name
(Required)

Enter the host name or the IP address of the Apache Solr server that you want to use.

Solr port
(Required)

Enter the port number that the Apache Solr server must use to communicate with the OGPL Web site. The default port used by Apache Solr is 8983

Solr path

Enter the path to location where you have installed the Apache Solr server.

Number of items to index per cron run

Click the maximum number of items that must be indexed in each cron job. For more information on cron, see <topic for cron>.

For example, if you want to index 200 items for each cron job, click 200.

Results per page

Enter the number of search results that must appear on the Search page of the OGPL Web site. For example, if you want to display 10 results per page, enter 10.

On Failure

Click the alternative result that must appear when the Apache Solr server is not available.

For example, if you want to display an error message when the Apache Solr server is not available, click Show error message.

Advanced Configuration

This section provides you additional options to configure the search functionality on the OGPL Web site

Extra help messages for administrators

If you enable help messages that provide more information to the administrators of the OGPL Web site, click Enabled.

Index write access

If you want the cron to send the updates made to the content to the Apache Solr server, click Read and write (normal).

Make Apache Solr Search the default

If you want to use the Apache Solr server as the default search server for the OGPL Web site, click Enabled.

Use Apache Solr for Taxonomy links

If you have used Taxonomy tags to group the content and if you want the Apache Solr to use these tags for searching the keyword, click Enabled.

Autocomplete widget to use

If you want to display the suggestions for the keyword when a visitor enters the search query, click Custom autocomplete widget. This allows visitors to enter a few characters and select the appropriate keyword from the suggested keywords.

If you want the visitor to press ENTER or click Search before displaying the search results, click Drupal core autocomplete widget.

Enable additional keyword suggestions on the autocomplete widget

If you want to provide related word suggestions for a search query below the search box while the visitor enters words in the search box, select this check box.

For example, if a visitor enters the word Technology in the search box, the word Technologies appears below the search box.

Enable spellchecker suggestions on the autocomplete widget

If you want to provide suggestions for the correct spelling of a word when a visitor enters a wrong spelling for a search query, select this check box.

For example, if a visitor enters Tecchnologies, the word Technologies appears.

  1. Click Save Configuration.
    The Apache Solr server is configured.
