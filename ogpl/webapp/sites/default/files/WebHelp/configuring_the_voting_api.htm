Configuring the Voting API

Visitors to the OGPL Web site can rate a dataset on the following criteria:

  • Quality
  • Accessibility
  • Usability

 

The Open Government Platform Control Panel application computes the ratings for each of these criteria and displays the overall rating for the dataset, on the Raw Dataset, Apps, Documents, Tools, and Services pages of the OGPL Web site. Overall rating is also shown in Metric reports.

You can define the vote counting process in the voting module of the OGPL Control Panel.

To configure the voting API:

  1. In the Drupal menu, point to Site configuration, and click Voting API.
    The Voting API page appears.

  2. In the Anonymous vote rollover list, click the time period after which two votes from the same computer is considered valid.
    Notes:

    • Clicking Never will prohibit all computers from double-voting. The OGPL Control Panel application does not register votes that are cast for the second time, for the same dataset from publicly accessed computers, such as Internet cafes and hotels.

    • The Immediately option does not work because currently there is a limitation in the Voting API.

  3. In the Vote tallying section, click the appropriate option. The following table lists each field and its description.

Table: Description of options in the Vote tallying section

Title

Description

Tally results whenever a vote is cast

Votes are counted in real-time.

Tally results at cron-time

Votes are counted when the cron job is executed. This option is best used for Web sites with high traffic.

Do not tally results automatically: I am using a module that manages its own vote results

Does not count votes.

A different module is used for counting votes.

  1. Click Save configuration.
    The voting API in the OGPL Web site is updated.
