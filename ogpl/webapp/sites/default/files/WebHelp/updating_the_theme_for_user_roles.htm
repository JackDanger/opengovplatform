Updating the Theme for User Roles

If you want to separate the Drupal administrative interface from the front-end interface, you can assign separate themes for each user role, including anonymous users. The Role Theme Switcher overwrites the default theme that the CMS Administrator had set in the Manage Themes section. For more information, see Managing Themes.

To change the default theme for each user role:

  1. In the Drupal menu, point to User management, and then click Role theme switcher.
    The Role Theme Switcher page appears.

  2. Click the theme that you want to assign for each user role. For example, if you want to set the dms theme as the default theme for CMS Administrators, in the Cms administrator list, click dms.
    Note: If you click Default, the default theme that the CMS Administrator or the Super Administrator had set in the Manage Themes section is applied. For more information, see Managing Themes.

  3. Click Save Configuration.
    The theme configuration is applied to each user role.
