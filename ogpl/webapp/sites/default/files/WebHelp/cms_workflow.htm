Understanding the Content Management System (CMS) Workflow

The Content Management System (CMS) workflow defines the process to create, review, and publish content on the OGPL Web site.

When new content is required, the CMS Administrator creates a content to address the requirement. CMS administrator can create the following content types for the OGPL application:

  • Block. For more information, see  Creating Blocks.

  • Country data. For more information, see  Creating Country Data.

  • FAQ. For more information, see Creating FAQ.

  • Featured Gallery. For more information, see Creating Featured Gallery.

  • Page. For more information, see Creating Pages.

  • Rotating Panel. For more information, see Creating Rotating Panels.

 

For example, if a new FAQ is required, the CMS Administrator creates an FAQ content, and assigns a Content Creator and a Moderator to the FAQ. An entry to update the FAQ content appears in the content list of the Content Creator. The Content Creator updates the FAQ by adding the required information based on the guidelines that CMS Administrator sets, and then submits the content for review to the Moderator.

An entry to review the FAQ content appears in the content list of the Moderator. The Moderator reviews the FAQ content based on the guidelines that CMS Administrator sets. The Moderator can approve or reject the FAQ content. If the Moderator rejects the FAQ content, it goes back to the Content Creator for rework. If the Moderator approves the FAQ content, the FAQ content is sent to the Publisher. The Publisher can also accept or reject the FAQ content. If the Publisher rejects the FAQ content, the FAQ content goes back to the Content Creator or the Moderator for rework. If the Publisher approves the FAQ content, the FAQ content is published on the OGPL Web site.
