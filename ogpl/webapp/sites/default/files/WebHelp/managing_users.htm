Managing Users

A Content Management System (CMS) Administrator manages the content management workflow. The CMS Administrator assigns roles to users, creates content, and assigns a Content Creator to update the content, and assigns a Moderator to review the content.

The following table lists various user roles and their responsibilities.

Table: Description of User Roles and their responsibilities

Role

Description

Content Creator

  • Updates the content that a CMS Administrator assigns.

  • Specifies the expiry date for the content.

  • Specifies the number of days before the expiration date when the OGPL Control Panel application must send the expiry reminder e-mail notification.

  • Sends the updated content for review to the Moderator.

  • Updates content as per review comments from the Moderator or the Publisher.

  • Views content revisions.

  • Tracks content updates.

Moderator

  • Reviews the content that Content Creator submits for review.

  • Updates the expiry date for the content that the Content Creator submits.

  • Updates the number of days before the expiration date when the OGPL Control Panel application must send the expiry reminder e-mail notification.

  • Sends rejected content, with review comments, back to the Content Creator.

  • Sends the approved content to the Publisher.

  • Views content revisions.

  • Tracks content updates.

Publisher

  • Reviews content that the Moderator submits.

  • Sends the rejected content, with review comments, back to the Content Creator or the Moderator.

  • Views content revisions.

  • Tracks content updates.

  • Publishes the approved content on the OGPL Web site.
