Configuring the Content Bias Settings

When a visitor searches for a keyword on the OGPL Web site, the Apache Solr search server scans the search index and assigns a score to the matches found based on factors, such as keyword relevance and keyword density. You can provide additional weightage, also called as bias, to certain properties when ordering the search results so that the Apache Solr search server assigns a higher score to the search results that refer to those content types. Search results with higher scores are listed at the top of the list.

To configure content bias settings:

  1. In the navigation menu, point to Configurable Items, Manage Header Content, and then click Manage Global and Advance Search.
    The Settings page appears.

  2. Click the Content bias settings tab.
    The Apache Solr page appears and displays various biasing parameters.

  3. In the Result Biasing section, click the appropriate weightage for various content properties. The following table lists various content properties and their description.

Table: Description of fields in the Apache Solr page

Content Property

Description

Sticky at top of the list

Click the numeric value that you want to assign as bias to the nodes that you have set to appear at the top of the search results while creating them.

Promoted to home page

Click the numeric value that you want to assign as bias to the nodes that you have promoted to home page.

More recently created

Click the numeric value that you want to assign as bias to the nodes that are created more recently. This setting changes the result scoring so that these nodes appear before those with higher keyword matching.

More comments

Click the numeric value that you want to assign as bias to the nodes with maximum content. This setting changes the result scoring so that nodes with more comments appear before those with higher keyword matching.

More recent comments

Click the numeric value that you want to assign as bias to the nodes that have received comments recently from the visitors of the OGPL Web site. This setting changes the result scoring so that nodes with the most recent comments appear before those with higher keyword matching.

For example, the datasets for which the visitors provided comments more recently appear at the top of the search results.

  1. Click Save Configuration.
    The search results settings are updated according to the bias that you specified for each property.

  2. In the Type biasing and exclusion section, click the appropriate numeric value that you want to assign as bias for the various content types available on the OGPL Web site. For example, if you want to give additional bias to the content type FAQ than the content type Featured Gallery during a search, you can select 5 for the FAQ type content bias and 4 for the Featured Gallery type content bias.

  3. In the Type biasing and exclusion section, select the check box for each content type that you want to exclude from the search index. These content types do not appear in any search results. For example, if you do not want content for charts and country data to appear in any search results, select the check box for Charts and Country Data.

  4. Click Save Configuration.
    The search results settings are updated according to the bias that you specified for each content type and the content type that you specified to be excluded does not appear in any search results.
