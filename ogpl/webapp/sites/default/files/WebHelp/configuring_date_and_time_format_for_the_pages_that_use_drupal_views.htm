Configuring the Date and Time Format for Pages that Use Drupal Views

OGPL Web pages, such as \Suggested Datasets, Suggested Apps, Valuable suggestions and ideas, and Latest Pages In Last 7 Days, use Drupal Views to fetch and display data. The format of the date and time that appears on these pages are defined in the Views that these pages use as their content source. You can configure a View and specify the format of the date and time that must appear when a page users this view.

To configure the date and time format for a view:

  1. In the Drupal menu, point to Site building, Views, and then click List.
    The Views page appears and displays the list of views for different blocks of the OGPL Web site.

  2. Navigate to the content source for the page that you want to configure, and click the Edit link. For example, if you want to configure the date and time format for the Suggested Datasets page, navigate to the suggested_dataset view, and click the Edit link.
    The Edit View View_Name page appears, where View_Name is the name of the view that you selected.

  3. In the left pane, click the name of the view for which you want to update the date and time format. For example, if you want to update the date and time format for the Suggested Datasets view, click Suggested Datasets.
    The details of the view that you selected appear in the right pane.

  4. In the Fields section, click the Node: Post date link.
    The View_Name: Configure field Node: Post date section appears, where View_Name is the name of the view that you selected.

  5. Scroll down until you see the Custom date format box.

  6. In the Custom date format box, enter the format for the date and time that must appear on the page that uses this view as the content source. For example, if you want the page to display the date in dd-mm-yyyy format, in the Custom date format box, enter d-m-Y.
    Tip: If you want to view the list of available date formats, click the PHP docs link.

  7. Click Update.

  8. Click Save.
    The date and time format on the page that uses this view is updated.
