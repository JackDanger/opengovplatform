Sending Back Content

If the content that a content creator updates is not as per the requirements or the content it contains any errors, you can assign the content back to the content creator, the moderator or the publisher for further update. For example, if the content needs to be updated or re-written, you can assign it back to the Content Creator. If the content needs to be reviewed again, you can assign it back to the Moderator.

While sending the content back in the workflow, you can also view the workflow history for that content. The Workflow History section displays details, such as the current workflow state for this content, when the content was created, when you send the content back to the Content Creator, when the content was sent to a Publisher for approval, or how many times the content has gone back for updates.

 

To send the content back into workflow:

  1. In the navigation menu, click the Home link.

  2. In the Content List section, click the Send this record back in workflow icon for the content that you want to send back.
    A page with the title of the content that you selected appears.

  3. If you want to assign the content back to the Content Creator, in the Change Content Management state section, click Create/Edit Content.
    If you want to assign the content back to the Moderator for further review, in the Change Content Management state section, click Moderator Review.
    If you want to assign the content back to the Publisher for review and publication, in the Change Content Management state section, click Publisher Review.

  4. In the Comment box, enter your comments, such as the reason for sending the content back or suggestions about how the content can be improved.

  5. Click Submit.
    The content is sent back to the user that you specified.
